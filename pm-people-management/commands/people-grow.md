# /people-grow — People Development

Develop your people: write performance reviews, design career paths, create onboarding plans, and (when needed) build PIPs.

## Invocation

```
/people-grow performance review for Minh (Senior PM)
/people-grow career ladder for Product team
/people-grow onboarding plan for new data engineer
/people-grow pip for [person] — [context]
/people-grow                    # asks what you need
```

## Workflow

### Step 1: Understand the Need

Ask:
- What do you need? (review, career ladder, onboarding, PIP)
- Who is this for? (individual, role, team)
- What's the context? (review cycle, new hire, performance concern)

Route to the appropriate skill based on the answer.

### Route A: Performance Review

Apply the **performance-review** skill:

1. Gather accomplishments and challenges for the period
2. Identify strength patterns and growth areas
3. Write evidence-based feedback using SBI framework
4. Create forward-looking development plan
5. Connect to career goals

Output: Complete performance review document.

### Route B: Career Ladder

Apply the **career-ladder** skill:

1. Define levels with scope and impact expectations
2. Write behavioral expectations per competency per level
3. Create promotion criteria with evidence requirements
4. Design IC and Management tracks
5. Map to compensation philosophy

Output: Complete career ladder framework.

### Route C: Onboarding Plan

Apply the **onboarding-plan** skill:

1. Design pre-boarding setup
2. Plan Week 1 in daily detail
3. Define 30/60/90 day milestones
4. Assign buddy and key introductions
5. Create check-in cadence and success criteria

Output: Complete 30/60/90 day onboarding plan.

### Route D: Performance Improvement Plan

Apply the **pip-plan** skill:

1. Document specific performance gaps with evidence
2. Confirm prior feedback was given
3. Define measurable improvement targets
4. Design support structures (coaching, training)
5. Set timeline, checkpoints, and outcome paths

Output: Complete PIP document.

### Step 2: Offer Next Steps

- **After review**: "Want me to **create a development plan** based on the growth areas?"
- **After career ladder**: "Should I **write the JD** for each level?"
- **After onboarding**: "Want me to **design the Week 1 calendar** in more detail?"
- **After PIP**: "Should I **plan the weekly check-in structure**?"

## Notes

- Performance reviews should contain ZERO surprises — continuous feedback is the foundation
- Career ladders need stakeholder buy-in — socialize before launching
- Onboarding success is measured at 90 days, not Day 1
- PIPs should be genuine opportunities, not paper trails — invest in the person
- Vietnamese cultural context: balance direct feedback with face-saving awareness
